Design Principles

This schema filters information based on evidence quality and removes subjectivity, ensuring all extracted data is verifiable, auditable, and ML-ready.

Deterministic Extraction

No guessing, no assumptions. Every value is either measured, calculated, or marked unclear. Subjective interpretations are explicitly forbidden.

Full Traceability

Every numerical value includes its source: page number, table/figure ID, and text snippet. Enables cross-verification and audit trails for every data point.

ML-Ready Structure

Flat, queryable JSON structure with controlled vocabularies. Designed for direct ingestion into machine learning pipelines and cross-paper meta-analysis without manual reformatting.

Three-Tier Data Classification

The schema categorizes every field into Tier A (Core), Tier B (Valuable), or Tier C (Context) to prioritize extraction effort and assess paper completeness.

Tier A

Core / Critical

Essential for any analysis. Includes material identity, replacement levels, control reference, at least one performance metric, and test conditions.

Improves analysis depth. Chemical composition, particle size, processing details, microstructural analysis, and durability tests.

Tier C

Contextual / Supporting

Adds context but not critical. Geographic origin, novelty claims, author interpretations, and scalability discussions.

Waste Material Identity

Tier A (Core) • 35+ structured fields

Captures the precise identity of the waste material, including its origin, physical form, chemical composition, and batch-specific characteristics. This module ensures materials are uniquely identifiable and comparable across studies.

Core Extraction Fields

Primary Waste Material

Common name, author-used name, material category (e.g., olive_pomace, pomace_ash)

Material Form

Physical form: raw, dried, ground, powder, ash, biochar. State at use: as-received, pre-treated.

Origin & Source

Geographic origin (country, region), source facility type (olive mill, lab-prepared), extraction method (two-phase, three-phase).

Chemical Composition

Elemental analysis (C, H, N, O), proximate analysis (moisture, ash, volatiles), oxide composition (SiO₂, Al₂O₃, CaO).

Physical Properties

Particle size distribution (d10, d50, d90), bulk density, specific gravity, specific surface area.

Formulation & Mix Design

Tier A (Core) • 40+ structured fields

Documents how the waste is integrated into the host material system, including the matrix type, substitution strategy, dosage levels, and mix proportions. Essential for understanding performance trends vs. waste loading.

Host Matrix System

Matrix category: Cement-based, Clay-based, Geopolymer, Composite
Specific system: Mortar, Concrete, Fired brick, Unfired brick, etc.
Application type: Structural, Non-structural, Insulation, Masonry

Role of Waste Material

Replacement
Substitutes cement, clay, sand, or aggregate
Addition
Added as filler or functional additive
Pore-former
Burns out during firing
Reinforcement
Fiber-based reinforcement

Substitution Levels

Control Reference (MANDATORY)

The baseline formulation (typically 0% waste) against which all variants are compared.

Variant Levels

All tested substitution percentages (e.g., 5%, 10%, 15%, 20%)

Pre-Treatment & Processing

Tier B (Valuable) • 45+ structured fields

Captures all energy-intensive processing steps applied to the waste before or during material production. Critical for assessing environmental impact, cost, and scalability.

Thermal Processing

Drying: Temperature, duration, method
Calcination: Peak temp (°C), holding time, heating rate
Pyrolysis/Torrefaction: Temperature, atmosphere
Firing (for clay bricks): Kiln type, peak temp, firing curve

Mechanical Processing

Grinding: Mill type, duration, energy input
Sieving: Target particle size, sieve size
Crushing: Jaw crusher, ball mill, etc.
Mixing: Mixer type, mixing time

Chemical Processing

Acid washing: Chemical used, concentration, duration
Alkali activation: NaOH/KOH molarity, curing conditions
Surface modification: Silane treatment, surfactant use
Extraction: Solvent type, temperature, extraction time

Energy Intensity Indicators

The schema automatically flags high-energy processing steps to support environmental and economic assessment:

"high_temperature_step_present"

Condition marker: TRUE if any step exceeds 500°C (e.g., calcination at 700°C)

"mechanical_grinding_present"

Condition marker: TRUE if ball milling or fine grinding applied

"chemical_treatment_present"

Condition marker: TRUE if acid/alkali treatment used

"multi_step_processing"

Condition marker: TRUE if ≥2 processing types combined

Quantitative Results

Tier A (Core) • 60+ performance metrics

The results matrix is the core output module, capturing all measured or calculated performance metrics. Every value includes test age, standard reference, and data origin traceability.

Mechanical

  • Compressive Strength (MPa)
  • Flexural Strength (MPa)
  • Tensile Strength (MPa)
  • Modulus of Elasticity (GPa)
  • Hardness (Shore D, Vickers)

Physical

  • Bulk Density (kg/m³)
  • Water Absorption (%)
  • Porosity (open/total, %)
  • Shrinkage (drying/firing, %)
  • Specific Gravity

Thermal

  • Thermal Conductivity (W/mK)
  • Specific Heat Capacity (J/kgK)
  • Thermal Diffusivity (m²/s)
  • Fire Resistance (classification)
  • Thermal Expansion Coefficient

Durability

  • Freeze-Thaw Resistance
  • Acid/Sulfate Resistance
  • Chloride Penetration
  • Carbonation Depth (mm)
  • Weathering Resistance

Technology Readiness & Feasibility

Tier C (Context) • Scalability assessment

Evaluates whether the proposed solution is ready for real-world deployment. Captures Technology Readiness Level (TRL), scalability barriers, and commercialization constraints.

Technology Readiness Level (TRL) Scale

Basic Research
TRL 1-2

Basic principles observed, concept formulated

TRL 3

Proof of concept demonstrated in lab

Development Phase
TRL 4

Lab-scale validation, component testing

TRL 5-6

Pilot-scale demonstration, prototype testing

Deployment Ready
TRL 7-8

Demonstration in operational environment

TRL 9

Full commercial deployment proven

Identified Barriers

Energy Intensity

High-temperature processing (>800°C), long calcination times

Complex Processing

Multi-step treatments, specialized equipment requirements

Material Availability

Seasonal supply, geographic concentration, quality variability

Performance Gaps

Strength reduction at higher substitution levels, durability concerns

Readiness Enablers

Ambient Curing

No thermal treatment required, room temperature processing

Standard Equipment

Uses conventional concrete/brick production machinery

Drop-in Replacement

Direct substitution without formulation redesign

Proven at Pilot Scale

Industrial demonstrations conducted, field testing completed

Evidence-Based Quality Checks

The schema includes built-in quality filters to prevent low-quality data from contaminating the dataset. Every paper is automatically scored for completeness and reliability.

Minimum Viability Check

Papers must have: material identity, ≥1 substitution level, control reference, and ≥1 performance metric.

"tier_a_viable": true

Modeling Eligibility

For ML training: control + ≥2 variants, all with measured (not estimated) data.

"allowed_for_modeling": true

Zero-Subjectivity Rule

No guessed values. Unclear data must be marked as unclear or omitted.

"no_guessed_values": true
