PhD Data Methodology

Quantitative Evaluation:
LLMs for Scientific Extraction

An objective assessment of Large Language Models for transforming unstructured scientific literature into database-ready schema.

The SDES Methodology

Scientific Data Extraction Score: Measuring "PhD-Readiness"

SDES = DQ (40%) + SA (25%) + CU (20%) + PF (15%) − PH (Penalty)
DQ 40%

Data Quantity

Recall of explicit numerical values (e.g., 30 MPa).

Why? The number is the fundamental unit of information. Missed numbers = useless extraction.

SA 25%

Schema Adherence

Machine readability for Python/R scripts.

Why? Structure allows automation. Data in "comments" fields requires manual cleanup.

CU 20%

Contextual Logic

Semantic disambiguation (e.g., test type).

Why? Scientific terms are context-dependent. Mis-categorization ruins statistical models.

PF 15%

Process Fidelity

Metadata capture (e.g., @ 28 days, 20°C).

Why? A value without conditions (metadata) is scientifically meaningless.

PH Penalty

Hallucination

Deduction for false positives.

Why? Null data is better than fake data. Inventions contaminate the dataset.

Model Performance Landscape

S-Tier Gold Standard

Criteria: Flawless structure. Zero hallucinations.

Use: Automated Database Ingestion.

A-Tier Research Assistant

Criteria: High accuracy, minor formatting quirks.

Use: High-quality extraction + cleanup script.

B-Tier Reviewer

Criteria: Good summary/context, poor JSON/Schema.

Use: Literature screening/filtering.

C-Tier Apprentice

Criteria: Basic metadata only. Fails complex tables.

Use: Simple indexing.

F-Tier Failure

Criteria: Hallucinations (invented data).

Use: Unusable for science.

Detailed Insights & Feedback

S-Tier

"Correctly mapped numerical values directly to schema keys."

DeepSeek v2

A-Tier

Insight: Highest recall (DQ) but poor schema adherence (SA); values often placed in prose notes.

B-Tier

C/F-Tier: Failures

Gemini 3 Flash

C-Tier

Mistral & DeepSeek v1

F-Tier

Hallucination by association—misinterpreted keywords and invented data. Severe PH penalty applied.

Final Recommendation

For experimental rigor and data integrity, Gemini 3 Pro Preview is selected as the primary extraction engine (S-Tier).

  • Schema Adherence (SA): Dynamically aligns schemas with scientific domains for Python parsing.
  • Data Quantity (DQ): Highest ratio of structured numerical pairs.
  • Process Fidelity (PF): Precise capture of operational parameters (metadata).

Suggested Citation Note

"While models such as ChatGPT (GPT-5.2) and DeepSeek v2 exhibited high data recall, they lacked the schema flexibility required for automated database construction, frequently relegating critical numerical data to unstructured text fields."
